## üìù Speaker Notes Guidelines

Every slide includes:
- **What to say** - Scripted talking points
- **What to show** - Live demo steps (if applicable)
- **What to check** - Student understanding checks
- **Time estimate** - How long to spend on slide
- **Transition** - Link to next slide

### Example Speaker Note Format:
```
[SLIDE 15: Creating Navbar Component]

WHAT TO SAY:
"Now we're going to create our first component. Think of components 
like LEGO blocks - small, reusable pieces."

WHAT TO SHOW:
1. Right-click on components folder
2. Create new file: Navbar.js
3. Type the component code
4. Save and check browser

WHAT TO CHECK:
"Raise your hand if you can see the components folder in VS Code"

TIME: 5 minutes

TRANSITION:
"Great! Now that we have the component, let's add it to our layout..."
```
