## üí° Teaching Tips & Best Practices

### **Before Class:**
1. **Always test live demos** before class
2. Have backup examples ready
3. Prepare error scenarios
4. Know common pitfalls
5. Practice analogies out loud

### **During Lecture:**
1. **Show, don't just tell** - Live coding is critical
2. Use the analogies (LEGO blocks, apartment building)
3. Check for understanding frequently
4. Celebrate small wins
5. Normalize struggle and errors

### **During Lab Time:**
1. **Walk around constantly** - Don't sit at desk
2. Help students debug (don't fix for them)
3. Point to error messages
4. Encourage peer helping
5. Take notes on common issues

### **After Class:**
1. **Update FAQ** with new questions
2. Post solutions in Slack
3. Identify struggling students
4. Adjust next day's pacing
5. Celebrate progress

---

## üö® Common Pitfalls to Avoid

### **For Instructors:**
1. ‚ùå **Don't skip live demos** - Students need to see it work
2. ‚ùå **Don't assume knowledge** - Explain every step
3. ‚ùå **Don't rush through errors** - Errors are teaching moments
4. ‚ùå **Don't code too fast** - Students need time to type
5. ‚ùå **Don't ignore lab time** - Circulate and help

### **For Students (warn them about):**
1. ‚ùå **Not reading error messages** - Errors tell you what's wrong!
2. ‚ùå **Not saving files** - Ctrl+S constantly!
3. ‚ùå **Not committing often** - Commit every 30 min
4. ‚ùå **Waiting until last minute** - Start early!
5. ‚ùå **Not asking for help** - 15-minute rule!
